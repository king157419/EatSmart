## 2026-03-04 Session 4 - React Hooks Error Fix

### ✅ Critical Bug Fixed

**Problem**: React Hooks error after UI improvements implementation:
```
Error: Rendered fewer hooks than expected. This may be caused by an accidental early return statement.
at Home (src/app/page.tsx:332:9)
```

**Root Cause**:
- `useSwipeable` hook was called inside `.map()` loops in `HistoryModal.tsx` (lines 46 and 85)
- This violates React's Rules of Hooks: hooks must be called at the top level, not inside loops, conditions, or nested functions
- When `meals.length` or `exercises.length` changed, the number of hook calls changed, breaking React's internal tracking

**Solution**: Created separate `SwipeableListItem` component
- Extracted swipeable list items into independent component
- Hook is now called at component top level (compliant with React rules)
- Each list item is a separate component instance with its own hook

### Technical Changes

**Files Created**:
- `frontend/src/components/Modals/SwipeableListItem.tsx` (45 lines)
  - Encapsulates `useSwipeable` hook
  - Handles swipe-to-delete gesture

**Files Modified**:

  - Removed internal `useSwipeable` calls from map loops
  - Removed `useState` for swipe tracking (moved to child component)
  - Replaced inline swipeable `<li>` with `<SwipeableListItem>` component
  - Reduced from 118 lines to 73 lines

## 2026-03-04 Session 2 - Bug Fixes After Feature Implementation

### ✅ Critical Bugs Fixed

**Problem**: After implementing F25-F28, backend API endpoints had bugs preventing proper functionality.

**Root Causes Identified**:
1. **Route Ordering Issue**: `/api/chat/session/today` was placed AFTER `/api/chat/session/{session_id}`, causing FastAPI to match "today" as a session_id parameter, resulting in 404 errors
2. **Missing Header Import**: `Header` was not imported from FastAPI, causing admin authentication to fail
3. **Header Parameter Issue**: Header parameter needed explicit alias to match HTTP header name

**Solutions Applied**:
1. Moved `/api/chat/session/today` route (line 532-539) to BEFORE `/api/chat/session/{session_id}` (line 495-502) in `backend/main.py`
2. Added `Header` to FastAPI imports: `from fastapi import FastAPI, HTTPException, Header`
3. Added alias to Header parameter: `x_admin_password: str = Header(None, alias="X-Admin-Password")`

### Verification Results

**Backend API Tests** (All Passing ✅):

**Modified Files**:

  - Line 10: Added `Header` to imports

### Key Learnings

1. **FastAPI Route Ordering**: Specific routes MUST come before parameterized routes, otherwise the parameterized route will match first
2. **FastAPI Header Parameters**: Need explicit `alias` parameter when HTTP header name contains hyphens (e.g., `X-Admin-Password`)
